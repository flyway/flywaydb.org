Website: Submit your Changes

Raise an issue

The first step is to raise an issue in the flywaydb.org Issue Tracker. This issue will be the place where you will be able to communicate with Flyway's committers and other users. You can use it to present your ideas and gather feedback.

Fork the repository

The next step is to fork the flywaydb.org GitHub repository:

Write awesome docs

Now is the time to work your magic and write stuff! Commit your changes to your fork and don't forget to pull and merge the changes from the main repo as they happen.

To view your changes locally you can execute this command in the root of your working copy:

> jekyll serve -w
