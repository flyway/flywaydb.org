Command-line tool

The Flyway command-line tool is meant for users who

  • do not run their applications on the JVM
  • wish to migration their database from the command-line without having to install Maven or Ant

Usage

flyway [options] command

Commands

Name Description
clean Drops all objects in the configured schemas
init Creates and initializes the metadata table
migrate Migrates the database
validate Validates the applied migrations against the ones available on the classpath
info Prints the details and status information about all the migrations
repair Repairs the metadata table after a failed migration

Configuration

The Flyway Command-line tool can be configured in the following ways:

Configuration file

flyway.user=myuser
flyway.schemas=schema1,schema2
flyway.placeholders.keyABC=valueXYZ

By default, the configuration will be read from conf/flyway.properties.

To use an alternative configuration file use -configFile=myAlternativeConfig.properties
To use an alternative configuration file encoding (default: UTF-8) use -configFileEncoding=ISO-8859-1

Command-line parameters

flyway -user=myuser -schemas=schema1,schema2 -placeholders.keyABC=valueXYZ migrate

Overriding order

Command-line parameters > Configuration file

Debug output

Adding -X to the argument list with make the command-line tool also print debug output.
